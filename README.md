### 📦 Bag Check v4 (第8回課題：価値検証バージョン)

#### 1. v4で追加した機能
- **マイテンプレート機能**: 自分用に荷物リストを調整し、独自のテンプレート名で保存・ロード・削除できます。
- **「今日も忘れ物なし」記録機能**: 最終確認を終えて出発する際、日付とともに「忘れ物なし！」という成功体験をログ履歴として蓄積します。
- **連続チェック日数表示 (Streak)**: 毎日お出かけチェックを継続すると、炎のアイコンとともに「連続 N 日チェック中！」のカウンターが上昇します。
- **価値検証用アンケート機能**: アプリ下部に「継続利用の評価（4段階）」と「理由・改善点」を入力できるフォームを設置。登録されたフィードバックはリアルタイムで下部に一覧表示され、価値検証テストの記録として活用できます。
- **UIと導線の改善**: 忙しい朝でも迷わない「3ステップガイド（出発設定 ➔ 荷物チェック ➔ 最終安全確認）」を最上部に設置し、全体の進行度を視覚化しました。

#### 2. v3からv4への差分
- **v3**: 出発予定時刻、残り時間の計算、重要アイテム表示、持ち物状況による「忘れ物リスク：高・中・低」判定といった「出発前に何を確認すべきか」に焦点を当てた機能を追加。
- **v4**: v3の基本機能を維持しつつ、「毎日継続して使ってもらう」ためのマイテンプレート、連続日数カウント、履歴表示を追加。さらに、他人に使ってもらった結果をアプリ内で直接回収できるフィードバック欄を新たに構築しました。

#### 3. 価値検証で確認したいこと
- 被験者が「出発予定時刻」や「残り時間」を目にすることで、実際に忘れ物への注意力が向上するか。
- 「連続チェック日数（Streak）」や「忘れ物なし記録」があることで、被験者がお出かけ前に「アプリを起動してチェックする」という継続的な習慣を定着させられるか（ゲーミフィケーションの有効性検証）。
- マイテンプレート機能により、デフォルトにない持ち物を容易に設定・復元できることで、ユーザーが自分仕様のアプリとして「使いやすい」と感じるか。

#### 4. 今後改善したい点 (最終発表会へ向けたロードマップ)
- **スマート提案（AIアシスト）**: 天気予報やカレンダーの予定から、当日に必要な特有の持ち物（傘、ハンカチ、特定の教科書など）をAIが自動提案する機能。
- **音声アシスト**: 出発前の忙しい時間帯に、画面を注視せずとも音声で「財布チェックした？」「鍵持った？」と対話的に確認できる機能。
- **通知連携**: 出発時間までにチェックが終わっていない場合、スマホにプッシュ通知でアラートを送るリマインダー連携。

## 受講生用テンプレートの使い方

### 第3回(初回・Use this template)

1. このリポジトリの **緑の "Use this template" ボタン** で自分のリポジトリを作成
   - リポ名: `[DiscordID]-web3ai-2026` / Public
2. 自分のリポジトリの [`lectures/lecture3/README.md`](./lectures/lecture3/) を開く
3. そこに常設の **Antigravity 用固定プロンプト** をコピペ → 内容を埋めて Antigravity で実行
4. 自動で `lectures/lecture3/` の3ファイル更新 + commit + push
5. ポータルアプリ(第3回 課題提出欄)に GitHub レポURL を提出

### 第4回以降(累積運用)

1. **同じリポジトリ** を Antigravity で開く(Use this template は不要)
2. このリポジトリの `lectures/lecture[N]/README.md` から **第N回プロンプト** をコピペ
3. プロンプトが pub-docs から第N回フォルダを fetch → 自リポに `lectures/lecture[N]/` 追加 → 内容反映 → push
4. 同じレポURLをポータルで提出

→ **学生は「README からプロンプトをコピペ」だけ**(毎回同じパターン・新規レポ作成不要)
